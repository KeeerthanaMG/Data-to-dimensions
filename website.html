Understanding the Fundamentals

Before we dive into building our 3D visualizer, let's understand the core technologies and concepts we'll be using. Don't worry - we'll explain everything in simple terms!

1. React - Building Blocks of Modern Web üèóÔ∏è

What is React?

React is a JavaScript library created by Facebook for building user interfaces. Think of it like LEGO blocks - you create small, reusable pieces (called components) and combine them to build complex applications.

Real-world analogy: Imagine building a house. Instead of building everything from scratch each time, you have pre-made doors, windows, and walls. In React, these are "components" - reusable pieces of UI that you can use anywhere in your app!

Key React Concepts You'll Use:

  • Components: Reusable pieces of UI (like buttons, forms, cards). Think of them as custom HTML tags.
  • Props: Data passed from parent to child components (like function parameters).
  • State: Data that changes over time within a component (like variables that trigger re-renders).
  • Hooks: Special functions that let you use React features (useState for state, useEffect for side effects).
  • JSX: HTML-like syntax in JavaScript. Write <div>Hello</div> directly in JS!

2. Three.js - The 3D Graphics Engine ‚ú®

What is Three.js?

Three.js is a powerful JavaScript library that makes it easy to create 3D graphics in web browsers. It uses WebGL under the hood but provides a much simpler API. Think of it as the engine that powers 3D visualizations on the web.

You've seen Three.js in action: Google Maps 3D buildings, interactive product viewers on e-commerce sites, 3D data visualizations in news articles - all powered by Three.js or similar libraries!

Core Three.js Building Blocks:

  • Scene: The 3D world container - like a stage where all your 3D objects live.
  • Camera: Your viewpoint into the scene. Determines what you see and from which angle.
  • Renderer: Takes the scene and camera and draws everything to the screen.
  • Mesh: A 3D object = Geometry (shape) + Material (appearance/color).
  • Geometry: The shape of your object (cube, sphere, or custom shapes).
  • Material: How the surface looks (color, texture, shininess).
  • Light: Illuminates your scene (ambient, directional, point lights).

3. React Three Fiber - React Meets Three.js üéØ

Why React Three Fiber?

Writing Three.js code directly involves a lot of boilerplate and imperative code. React Three Fiber (@react-three/fiber) is a React renderer for Three.js that lets you build 3D scenes using React components - making it much more intuitive and maintainable!

The difference:
Pure Three.js: const mesh = new THREE.Mesh(geometry, material); scene.add(mesh);
React Three Fiber: <mesh><boxGeometry /><meshStandardMaterial /></mesh>
Much cleaner and more React-like!

Benefits:

  • Write Three.js using familiar React components and JSX
  • Automatic memory management and cleanup
  • Use React's state, props, and hooks with 3D objects
  • Easier to reason about and maintain

4. Geographic Data Formats üó∫Ô∏è

When working with maps and geographic data, you'll encounter several data formats. Let's understand each one:

üìç GeoJSON

What: JSON format for encoding geographic data structures (points, lines, polygons).

Use Case: Web mapping, storing geographic features with properties.

Example: Building footprints, city boundaries, points of interest.

{"type": "Point", "coordinates": [lon, lat]}
üóÇÔ∏è Shapefile

What: Popular geospatial vector data format developed by Esri.

Use Case: GIS software, professional mapping applications.

Note: Consists of multiple files (.shp, .shx, .dbf). Can be converted to GeoJSON for web use.

üåê KML/KMZ

What: XML-based format used by Google Earth.

Use Case: Google Earth visualizations, sharing geographic data.

Note: KMZ is compressed KML. Great for complex visualizations with styling.

üé® TopoJSON

What: Extension of GeoJSON that encodes topology.

Use Case: More efficient than GeoJSON for large datasets.

Benefit: Smaller file sizes, preserves shared boundaries between shapes.

5. GeoJSON Deep Dive üìê

Since we'll be using GeoJSON in our project, let's understand its structure in detail:

Basic GeoJSON Structure:

{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": {
        "type": "Polygon",
        "coordinates": [[[lon1,lat1], [lon2,lat2], [lon3,lat3], [lon1,lat1]]]
      },
      "properties": {
        "name": "Building A",
        "height": 25,
        "color": "#ff0000"
      }
    }
  ]
}

GeoJSON Geometry Types:

  • Point: Single location (e.g., a marker on a map)
  • LineString: Series of connected points (e.g., a road or path)
  • Polygon: Closed shape with area (e.g., building footprint, park)
  • MultiPoint: Multiple separate points
  • MultiLineString: Multiple separate lines
  • MultiPolygon: Multiple separate polygons
Coordinate Order in GeoJSON: Always [longitude, latitude] - opposite of what you might expect! Think "X, Y" where longitude is X (horizontal) and latitude is Y (vertical).

6. PostGIS - Spatial Database Superpowers üóÑÔ∏è

What is PostGIS?

PostGIS is a spatial database extension for PostgreSQL. It adds support for geographic objects, allowing you to run location-based queries directly in your database. Think of it as a supercharged PostgreSQL that understands geography!

Why PostGIS matters: Instead of loading huge geographic datasets into your application and processing them in JavaScript, you can query and filter them efficiently in the database. For example, "Find all buildings within 5km of this point" runs instantly on millions of records!

What PostGIS Can Do:

  • Store geographic data types (points, lines, polygons) natively
  • Perform spatial queries: distance, intersection, containment
  • Calculate areas, lengths, and other geometric measurements
  • Transform between different coordinate systems
  • Export data as GeoJSON directly from SQL queries
Example PostGIS Query:
SELECT *, ST_AsGeoJSON(geometry) FROM buildings WHERE ST_Distance(geometry, ST_MakePoint(lon, lat)) < 1000;
This finds all buildings within 1000 meters of a point and exports them as GeoJSON!

7. Coordinate Reference Systems (CRS) üåç

What is a CRS?

A Coordinate Reference System defines how coordinates map to locations on Earth. Since Earth is a 3D sphere but we often work with 2D maps, we need different systems for different purposes.

Common CRS You'll Encounter:

WGS84 (EPSG:4326)

Most Common! Used by GPS, Google Maps, GeoJSON standard.

Coordinates: Latitude/Longitude in degrees

Range: Lat [-90, 90], Lon [-180, 180]

Web Mercator (EPSG:3857)

Used by web mapping (Google Maps, OpenStreetMap tiles)

Coordinates: Meters from origin

Good for: Web map tiles

UTM Zones

Universal Transverse Mercator - regional projections

Good for: Accurate local measurements

Used in: Surveying, engineering

For our project: We'll use WGS84 (standard GeoJSON) coordinates. Most GeoJSON data you find online already uses this system!

8. From 2D to 3D - The Extrusion Process üè¢

How do we turn flat GeoJSON into 3D buildings?

The process is called "extrusion" - we take a 2D polygon (building footprint) and "pull it up" to create height. Think of it like pushing cookie dough through a cookie cutter to create 3D shapes!

The Steps:

  • Read GeoJSON polygon coordinates (array of [lon, lat] points)
  • Convert to THREE.Shape (2D shape in Three.js)
  • Use ExtrudeGeometry with a "depth" parameter (this is the height)
  • Wrap in Material to give it color/texture
  • Create Mesh and add to Scene

9. 3D Visualization Best Practices üé®

Creating effective 3D visualizations requires more than just technical knowledge. Here are key principles:

Performance Considerations:

  • Level of Detail (LOD): Show simpler geometry when far away, detailed when close
  • Instancing: Reuse geometry for repeated objects (like identical buildings)
  • Culling: Don't render objects outside camera view
  • Texture Atlases: Combine multiple textures to reduce draw calls

Visual Design:

  • Color Encoding: Use color to represent data (height, temperature, population density)
  • Lighting: Proper lighting makes 3D depth more obvious - use ambient + directional lights
  • Camera Angles: Default to a 45¬∞ angle for best depth perception
  • Scale: Keep proportions realistic or your visualization will look wrong

Create Ground Plane

src/components/GroundPlane.jsx

This creates a flat ground surface so our 3D buildings have something to stand on.

function GroundPlane() {
  return (
    <mesh rotation={[-Math.PI / 2, 0, 0]} receiveShadow>
      <planeGeometry args={[200, 200]} />
      <meshStandardMaterial color="#e0e0e0" />
    </mesh>
  );
}

export default GroundPlane;
üîç What's happening here?

<mesh>: A 3D object = geometry (shape) + material (appearance)

rotation={[-Math.PI / 2, 0, 0]}: Rotates plane 90¬∞ to make it horizontal (floor)

-Math.PI / 2: That's -90 degrees in radians (Three.js uses radians)

receiveShadow: This surface can receive shadows from other objects

<planeGeometry args={[200, 200]}: Creates a flat square 200x200 units

<meshStandardMaterial>: Gives the plane color and realistic lighting response

color="#e0e0e0": Light gray color (hex color code)

6

Update Scene to Load GeoJSON

src/components/Scene.jsx

Now we'll modify our Scene to load the GeoJSON file and render buildings.

import { useEffect, useState } from 'react';
import Building from './Building';
import GroundPlane from './GroundPlane';
import { Canvas } from '@react-three/fiber';
import { OrbitControls } from '@react-three/drei';

function Scene() {
  const [features, setFeatures] = useState([]);

  useEffect(() => {
    fetch('/src/data/buildings.geojson')
      .then(res => res.json())
      .then(data => setFeatures(data.features));
  }, []);

  return (
    <Canvas camera={{ position: [0, 50, 100], fov: 60 }}>
      <ambientLight intensity={0.5} />
      <directionalLight position={[10, 20, 10]} intensity={1} />
      <GroundPlane />
      {features.map((feature, idx) => (
        <Building key={idx} feature={feature} />
      ))}
      <OrbitControls />
    </Canvas>
  );
}

export default Scene;
üîç New Concepts:

useState: React hook to store component state (data that changes)

const [features, setFeatures]: Creates state variable "features" and function to update it

useState([]): Initial value is empty array

useEffect: Runs code after component mounts (side effects)

fetch: Loads the GeoJSON file from the specified path

.then(res => res.json()): Converts response to JavaScript object

setFeatures(data.features): Updates state with GeoJSON features

[], []: Empty dependency array - run once when component loads

features.map: Loop through each feature and create a Building component

key={idx}: Unique identifier for each Building (React requirement)

7

Create Building Component

src/components/Building.jsx

This is where the magic happens! We convert 2D GeoJSON polygons into 3D buildings.

import * as THREE from 'three';

function Building({ feature }) {
  const { coordinates } = feature.geometry;
  const height = feature.properties.height || 10;

  // Convert GeoJSON polygon to THREE.Shape
  const shape = new THREE.Shape();
  coordinates[0].forEach(([x, y], i) => {
    if (i === 0) shape.moveTo(x, y);
    else shape.lineTo(x, y);
  });

  return (
    <mesh position={[0, 0, 0]}>
      <extrudeGeometry args={[shape, { depth: height, bevelEnabled: false }]} />
      <meshStandardMaterial color="#8bc34a" />
    </mesh>
  );
}

export default Building;
üîç Deep Dive - How 2D Becomes 3D:

import * as THREE: Imports entire Three.js library as "THREE" object

{ feature }: Destructures props - extracts feature from props object

feature.geometry.coordinates: Gets array of [x,y] points from GeoJSON

feature.properties.height || 10: Uses height property or defaults to 10

new THREE.Shape(): Creates a 2D shape object (like a cookie cutter)

coordinates[0]: First array is outer ring (GeoJSON can have holes)

forEach(([x, y], i): Loop through each coordinate point

shape.moveTo(x, y): Move to first point (don't draw yet)

shape.lineTo(x, y): Draw line from previous point to this point

<extrudeGeometry>: Takes 2D shape and "pulls it up" to create 3D!

depth: height: How far to extrude (building height)

bevelEnabled: false: No rounded edges (keep it sharp)

color="#8bc34a": Nice green color for buildings

üêõ Troubleshooting Common Issues

Issue: Black screen / Nothing renders

Solution:

  • Check browser console for errors (F12)
  • Make sure all files are in correct folders
  • Verify imports are correct (check file paths)
  • Ensure camera position is not inside an object

Issue: Buildings not showing

Solution:

  • Check GeoJSON file path in fetch()
  • Open Network tab in browser - is the file loading?
  • Console.log(features) to see if data is loaded
  • Try zooming out - buildings might be too small or far

Issue: Can't rotate/zoom

Solution:

  • Make sure OrbitControls is imported from @react-three/drei
  • OrbitControls must be inside <Canvas> tags
  • Check that drei is installed: npm install @react-three/drei

Issue: Performance is slow

Solution:

  • Reduce number of buildings for testing
  • Simplify geometry (fewer points in polygons)
  • Use simpler materials
  • Consider LOD (Level of Detail) for many objects
